# ファイル削除は rm ではなく trash を使う

ファイルやディレクトリを削除するときは、不可逆な `rm` 系ではなく `trash`（macOS のゴミ箱へ移動。復元可能）を使う。これは構文ではなく「削除する」という**意図**に適用する。

## 必須フロー

1. 削除には `trash <ファイル/ディレクトリ>` を使う。ディレクトリもそのまま渡せ、`-r` / `-f` は不要。
2. 次のような不可逆な削除手段は使わない（同じ意図は trash に置き換える）: `rm` / `sudo rm` / `/bin/rm` / `command rm` / `xargs rm` / `find … -delete` / `unlink`。
3. trash で代替できない完全削除が本当に必要な場合は、実行前に理由を添えてユーザーに確認する。

## 例外

- `git rm`（Git インデックスの操作）。
- ビルド / パッケージツールが内部で行う削除（`cargo clean`、パッケージマネージャのキャッシュ削除など）。
- ユーザーが明示的に完全削除を指示した場合。
