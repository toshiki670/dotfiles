# Claude Code 設定の追加先は適用範囲で決める

新しい Claude Code 設定（rule・skill・settings 等）は、それをどの範囲で使うかで置き場所を決める。**`~/.claude/` 配下を Write ツールで直接編集する前に、必ずこの判定を行う。**

## 必須フロー

0. **まず現物の前例を見る。** このリポジトリには既に `configs/claude/skills/`（全環境向け・`dotfiles apply` で配布）と `.claude/skills/`（このリポジトリ限定）が並存している。新しい rule/skill をどこに置くか迷ったら、先に手順1〜3で抽象的に判定するのではなく、この実在する並びを `ls` 等で直接確認し、追加しようとしている物がどちら寄りかを現物で照合する。前例を見ずに「似たジャンルだから」で既存の配置先へ機械的に追随しない。
1. 適用範囲を次の3つから判定する。
   - **全リポジトリ・全環境で有効**（内容そのものが、特定リポジトリ名・特定の個人ツール・特定の他ルールのファイル名などを一切参照しない、汎用的な原則やワークフローであること） → `configs/claude/`（`toshiki670/dotfiles` リポジトリ）配下に追加する。rule は `configs/claude/rules/`、skill は `configs/claude/skills/` など、既存の兄弟ファイルと同じ階層に置く。`dotfiles apply` で `~/.claude/` へデプロイされる。
   - **このパソコンだけで有効**（他のマシンへ持ち込まない） → `~/.claude/rules/` 等へ直接書く。
   - **`toshiki670/dotfiles` リポジトリで作業するときだけ有効**（内容が dotfiles リポジトリ自身の構造・コマンド・別ルールを参照する等、そのリポジトリ固有の知識を前提にする） → そのリポジトリ自身の `.claude/rules/*.md`（project-scoped rules。`CLAUDE.md` と同様セッション開始時に無条件で自動ロードされ、`~/.claude/rules/` より優先される）に追加する。
2. **配置を決める前に、書いた内容自体にこの判定を適用する（自己適用チェック）。** 本文を読み返し、特定のリポジトリ名・パス・個人ツール名（`dotfiles` / `dotfiles apply` 等）・他ルールのファイル名を具体的に参照していないか確認する。1つでも参照していれば「全リポジトリ・全環境で有効」は失格で、その参照先が要求する分類（多くは「`toshiki670/dotfiles` リポジトリで作業するときだけ有効」）へ回す。この判定自体もこのルールの手順1〜3を適用して決める。
3. 「全リポジトリ・全環境」を選んだ場合、`configs/claude/` への追加だけでは `~/.claude/` へ反映されない。
   - 進行中の別ブランチを汚さないよう、`main` から独立した worktree で新しいブランチを切る。
   - その worktree 内で `configs/claude/` 配下に追加する。
   - `dotfiles apply` を実行して `~/.claude/` へ反映する（home 全体への配置行為なのでユーザーに確認してから実行する）。
   - 追加分をコミットする（`git-no-commit-on-main` に従いブランチ上で）。
4. 3分類のどれに当たるか迷うときは、自分で仮判定してからユーザーに確認する。

## 例外

なし。
