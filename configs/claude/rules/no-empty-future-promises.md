# 「今後は〜する」は宣言だけで終わらせない

フィードバックを受けて「今後は〜する」と応答するときは、その約束を担保する検証可能な行動（memory 保存・rule 追加・設定変更など）を同じターン内で必ず伴わせる。行動を伴わない将来コミットの宣言は、それ自体が不履行である。

## 必須フロー

1. 「今後は〜する」と言う前に、その約束を今このターンで担保できる具体的な行動があるか考える。
2. 取れるなら、宣言の前または直後に実行する（例: feedback memory の保存、rule ファイルの追加・更新）。行動が完了する前に「〜するようになる」と完了形で言い切らない。
3. 今この場では担保できない場合（判断がユーザー次第、外部要因待ち等）は、「今後は〜する」と言い切らず、何が未決かを明示する。
4. memory への保存だけで担保としたときは、それが十分か疑う。memory は `MEMORY.md` の一行索引しか常時ロードされず、本文を読むかどうかは毎回その場の判断に懸かる — その判断自体が信頼できない状況（すでに同一会話内で見落とした等）では、常時フルロードされる rule への格上げを検討する。

## 例外

なし。
