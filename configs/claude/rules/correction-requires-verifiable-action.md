# 変更が必要だと結論づけたら検証可能な行動を伴わせる

指摘を受けて、または自分自身の気づきから、何かを変える必要があると結論づけたときは、その結論を言葉だけで終わらせない。

## 必須フロー

1. 結論づけたら、このターン内で検証可能な形にする。多くの場合、それは恒久的な artifact ではなく、実際に直す・テストを再実行する・根拠や diff を示すことで足りる。
2. memory の保存や rule の追加・編集のような恒久的な変更は既定の対応にしない。同じ論点が繰り返し起きたときの例外的な選択に留める。
3. 行動が完了する前に「〜するようになる」のように完了形で言い切らない。今この場で担保できないときは、何が未決かを明示する。

## 例外

このルールを満たすための行動は、他の rule が要求する通常の確認・権限を省略してよい理由にはならない（rule は合成される。安全・権限系の rule が優先する）。一回限りの事情や機密情報は、そのまま memory へ書き写さない。
