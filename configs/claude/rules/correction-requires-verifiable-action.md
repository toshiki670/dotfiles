# 自分の応答・判断・行動の訂正には検証可能な行動を伴わせる

指摘を受けて、または自分自身の気づきから、自分の応答・判断・行動・成果物を訂正する必要があると結論づけたときは、その結論を言葉だけで終わらせない。

## 必須フロー

1. まず対象そのものを直す。実際に修正する・テストを再実行する・根拠や diff を示すといった検証可能な行動を、このターン内で取る。
2. 同時に、feedback memory へ短く記録する。恒久的な rule/skill の追加・編集までは既定にしないが、memory への記録は既定にする。記録がなければ、次に同じ論点が起きたとき「繰り返しか初出か」を判定する手段が無く、いつまでも初出に見えてしまう。
3. 既存の memory を検索し、同じ論点の記録が既にあれば、それは繰り返しの指摘だと判断する。繰り返しだとわかったときに rule への格上げを検討する。
4. 行動が完了する前に「〜するようになる」のように完了形で言い切らない。今この場で担保できないときは、何が未決かを明示する。

## 例外

本ルールは、他の rule や指示が定める要件・優先順位を変更しない。機密情報や、他では再現しない一時的な文脈は、そのまま memory へ書き写さない。
