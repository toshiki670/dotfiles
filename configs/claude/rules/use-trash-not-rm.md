# ファイル削除は rm ではなく trash を使う

ファイルやディレクトリを削除するときは、不可逆な `rm` 系ではなく `trash`（macOS のゴミ箱へ移動。復元可能）を使う。これは構文ではなく「削除する」という**意図**に適用する。完全削除（不可逆な削除）は Claude 自身では行わず、ユーザーに委ねる。

## 必須フロー

1. 削除には `trash <ファイル/ディレクトリ>` を使う。ディレクトリもそのまま渡せ、`-r` / `-f` は不要。
2. `trash` へ渡す引数がコマンド置換や変数展開の結果であるときは、渡す前に非空であることを確認する（変数へ代入し `[ -n "$x" ]` 等で検証してから渡す）。空文字列を渡すとカレントディレクトリが対象になり、意図しないディレクトリごと移動する。PreToolUse フックは `trash "$(...)"` のような構文上検証漏れが疑われるパターンのみを検知するもので、変数を経由した実行時の空文字列までは追跡できない。`time` / `env` / `sudo` 等を経由して `trash` を呼ぶ形もフックの検知対象外なので、そもそもそうした経由をせず素直に `trash <ファイル/ディレクトリ>` の形で呼ぶ。
3. 次のような不可逆な削除手段は使わない（同じ意図は trash に置き換える）: `rm` / `sudo rm` / `/bin/rm` / `command rm` / `xargs rm` / `find … -delete` / `unlink`。
4. ユーザーが完全削除を望む場合も、Claude 自身は `rm` 等を実行しない。`trash` で移動したうえで「ゴミ箱を空にすれば完全削除できる」ことを案内するか、完全削除はユーザー自身に実行してもらう（PreToolUse フックも `rm` を常に止めるため、この方針と一致する）。
5. Git の追跡から外すときは、実体も削除する `git rm <path>`（デフォルト）は使わない。インデックスのみ外す `git rm --cached <path>` を使い、実体の削除が必要なら別途 `trash <path>` で行う。

## 例外

- ビルド / パッケージツールが内部で行う削除（`cargo clean`、パッケージマネージャのキャッシュ削除など）。
